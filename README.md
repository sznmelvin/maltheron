## Quick Start

### Prerequisites

- [Bun](https://bun.sh) installed (v1.3+)
- Node.js v18+ (for Convex CLI)
- Phantom Wallet (or compatible Solana wallet)

### 1. Install Dependencies

```bash
bun install
```

### 2. Set Up Convex

Initialize your Convex deployment:

```bash
npx convex dev
```

This will prompt you to log in to Convex and create a deployment.

### 3. Environment Variables

The `.env.local` file is automatically generated by `npx convex dev`. Ensure it contains:

```bash
CONVEX_URL=https://your-deployment.convex.cloud
VITE_CONVEX_URL=https://your-deployment.convex.cloud
NODE_ENV=development
LOG_LEVEL=debug

# Solana (for production)
SOLANA_TREASURY_WALLET=your_solana_wallet_address
```

### 4. Start Development Servers

```bash
bun run dev
```

- **API Core**: http://localhost:3000/v1
- **Ops Dashboard**: http://localhost:5173

## API Endpoints

### Authentication
- `POST /v1/auth/session` - Create session with wallet address
- `POST /v1/auth/dev/create` - Create dev agent (development only)
- `GET /v1/auth/me` - Get current agent info
- `POST /v1/auth/logout` - Invalidate session

### Ledger
- `POST /v1/ledger/transact` - Record transaction (0.1% fee deducted)
- `POST /v1/ledger/verify` - Verify on-chain transaction
- `GET /v1/ledger/recent` - Get recent transactions
- `GET /v1/ledger/agent/:id` - Get agent transactions
- `GET /v1/ledger/volume` - Get total transaction volume
- `GET /v1/ledger/fees` - Get total fee revenue
- `GET /v1/ledger/config` - Get Solana configuration

### Memory
- `POST /v1/memory/query` - Query financial memory (roi/spend_velocity/tax_liability)
- `GET /v1/memory/all` - Get all memory dimensions
- `GET /v1/memory/tax/stubs` - Get tax stubs
- `GET /v1/memory/tax/unpaid` - Get unpaid tax liability

### Agents
- `GET /v1/agents/:id` - Get agent by ID
- `GET /v1/agents` - List all agents
- `GET /v1/agents/stats/count` - Get active agent count

## Features

### Solana Integration
- **USDC Payments** - Native SPL token support on Solana
- **0.1% Fee** - Automatic fee collection to treasury wallet
- **Phantom Wallet** - Connect with popular Solana wallet
- **Devnet Ready** - Test with free devnet tokens

### Transaction Flow
1. Agent connects wallet (Phantom)
2. Agent sends USDC transaction on Solana
3. Backend verifies on-chain transaction
4. 0.1% fee deducted automatically
5. Transaction recorded in Convex
6. Real-time updates via Convex

### Financial Memory
- **ROI** - Return on investment calculations
- **Spend Velocity** - Daily/hourly spending patterns
- **Tax Liability** - Estimated tax based on 15% rate

### Real-time Dashboard
- Live transaction stream (polling every 3 seconds)
- API Playground for testing
- Memory query visualization
- Agent metrics display

## Transaction Fee

Maltheron deducts a 0.1% (10 basis points) fee on all transactions.

Example: 1000 USDC transaction → 999 USDC net + 1 USDC fee to treasury

## Production Deployment

### Environment Variables

For production, create a `.env` file with:

```bash
# Required
CONVEX_URL=https://your-production.convex.cloud
VITE_CONVEX_URL=https://your-production.convex.cloud

# Solana
SOLANA_TREASURY_WALLET=your_solana_wallet_address

# Security
NODE_ENV=production
LOG_LEVEL=info
ALLOWED_ORIGINS=https://your-domain.com
```

### Deploying to Render

1. Push your code to GitHub
2. Create a new Web Service on Render
3. Configure:
   - **Build Command**: `bun install && bun run build:backend`
   - **Start Command**: `bun run dist/index.js`
   - **Environment**: `Bun`
4. Add all environment variables

### Deploying to Vercel (Frontend)

1. Push your code to GitHub
2. Create a new project on Vercel
3. Configure:
   - **Framework**: Vite
   - **Build Command**: `bun run build`
   - **Output Directory**: `frontend/dist`
4. Deploy

### Convex Production

```bash
# Deploy Convex to production
npx convex deploy

# Update your CONVEX_URL to the production deployment
```
