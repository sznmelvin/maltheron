## Quick Start

### Prerequisites

- [Bun](https://bun.sh) installed (v1.3+)
- Node.js v18+ (for Convex CLI)
- Phantom Wallet (or compatible Solana wallet)

### 1. Install Dependencies

```bash
bun install
```

### 2. Set Up Convex

Initialize your Convex deployment:

```bash
npx convex dev
```

This will prompt you to log in to Convex and create a deployment.

### 3. Environment Variables

Copy `.env.example` to `.env.local` and configure:

```bash
# Convex (generated by npx convex dev)
CONVEX_URL=https://your-deployment.convex.cloud
VITE_CONVEX_URL=https://your-deployment.convex.cloud

# Solana Treasury (REQUIRED)
TREASURY_WALLET=SoLanaAddrHere11111111111111111111111111111111111111111
```

### 4. Start Development Servers

```bash
bun run dev
```

- **API Core**: http://localhost:3000/v1
- **Ops Dashboard**: http://localhost:5173

## Architecture

```
[Autonomous Agent]
   │
   ├── Send SOL to recipient (via Phantom)
   ├── Send 0.1% fee to treasury (via Phantom)
   │
   ▼
[Maltheron API] ─────────────────────────────────────┐
   │                                                  │
   ├── Verify main tx on Solana RPC                   │
   ├── Verify fee tx on Solana RPC                    │
   │                                                  │
   ▼                                                  ▼
[Convex Database] ◄─────────────────── [React Dashboard]
          │
          ▼
   [Financial Memory Engine]
          │
          └── ROI, Spend Velocity, Tax Liability
```

## Two-Transfer Flow

Maltheron is a **pure verification layer** - agents send transactions themselves via Phantom:

1. **Agent sends SOL** to recipient via Phantom
2. **Agent sends 0.1% fee** to treasury wallet via Phantom
3. **Agent submits main txHash** to `/v1/ledger/verify`
4. **Maltheron verifies** both transfers on-chain
5. **Maltheron records** in ledger if fee was paid
6. **Reject if fee missing** with "Fee not paid to treasury wallet"

### Example

```
Agent wants to pay 100 SOL to recipient:
├── Main transfer: 100 SOL → recipient
└── Fee transfer: 0.1 SOL → treasury (0.1% of 100 SOL)

Agent submits main txHash → Maltheron verifies both → Records 99.9 SOL net
```

## API Endpoints

### Authentication
- `POST /v1/auth/session` - Create session with wallet address
- `POST /v1/auth/dev/create` - Create dev agent (development only)
- `GET /v1/auth/me` - Get current agent info
- `POST /v1/auth/logout` - Invalidate session

### Ledger
- `POST /v1/ledger/verify` - Verify SOL transaction + fee payment
- `GET /v1/ledger/recent` - Get recent transactions
- `GET /v1/ledger/agent/:id` - Get agent transactions
- `GET /v1/ledger/volume` - Get total transaction volume
- `GET /v1/ledger/fees` - Get total fee revenue
- `GET /v1/ledger/config` - Get Solana configuration

### Memory
- `POST /v1/memory/query` - Query financial memory (roi/spend_velocity/tax_liability)
- `GET /v1/memory/all` - Get all memory dimensions
- `GET /v1/memory/tax/stubs` - Get tax stubs
- `GET /v1/memory/tax/unpaid` - Get unpaid tax liability

### Agents
- `GET /v1/agents/:id` - Get agent by ID
- `GET /v1/agents` - List all agents
- `GET /v1/agents/stats/count` - Get active agent count

## Transaction Fee

Maltheron requires a 0.1% (10 basis points) fee on all transactions.

**Fee is enforced on-chain:**
- Agent must send fee to treasury wallet
- API verifies fee transfer exists before recording
- Missing fee = transaction rejected

Example: 100 SOL transaction → 99.9 SOL net + 0.1 SOL fee to treasury

## Production Deployment

### Environment Variables

For production, configure:

```bash
# Required
CONVEX_URL=https://your-production.convex.cloud
VITE_CONVEX_URL=https://your-production.convex.cloud

# REQUIRED: Treasury wallet for fee collection
TREASURY_WALLET=your_solana_wallet_address

# Optional: Custom RPC (defaults to mainnet-beta)
# For production, use Helius or QuickNode:
# RPC_URL=https://mainnet.helius-rpc.com/?api-key=YOUR_KEY

# Security
NODE_ENV=production
LOG_LEVEL=info
ALLOWED_ORIGINS=https://your-domain.com
```

### Deploying to Render

1. Push your code to GitHub
2. Create a new Web Service on Render
3. Configure:
   - **Build Command**: `bun install && bun run build:backend`
   - **Start Command**: `bun run dist/index.js`
   - **Environment**: `Bun`
4. Add all environment variables

### Deploying to Vercel (Frontend)

1. Push your code to GitHub
2. Create a new project on Vercel
3. Configure:
   - **Framework**: Vite
   - **Build Command**: `bun run build`
   - **Output Directory**: `frontend/dist`
4. Deploy

### Convex Production

```bash
# Deploy Convex to production
npx convex deploy

# Update your CONVEX_URL to the production deployment
```
