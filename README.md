# Maltheron - Agent-Native Accounting & Financial OS

The financial operating system and memory layer for the M2M economy.

## Quick Start

### Prerequisites
- [Bun](https://bun.sh) installed (v1.3+)
- Node.js v18+ (for Convex CLI)

### 1. Install Dependencies

```bash
bun install
```

### 2. Set Up Convex

Initialize your Convex deployment:

```bash
npx convex dev
```

This will prompt you to log in to Convex and create a deployment.

### 3. Environment Variables

The `.env.local` file is automatically generated by `npx convex dev`. Ensure it contains:

```bash
CONVEX_URL=https://your-deployment.convex.cloud
VITE_CONVEX_URL=https://your-deployment.convex.cloud
BETTER_AUTH_SECRET=your-secret-key
NODE_ENV=development
LOG_LEVEL=debug
```

### 4. Start Development Servers

```bash
bun run dev
```

- **API Core**: http://localhost:3000/v1
- **Ops Dashboard**: http://localhost:5173

## Production Deployment

### Environment Variables

For production, create a `.env` file with:

```bash
# Required
CONVEX_URL=https://your-production.convex.cloud
VITE_CONVEX_URL=https://your-production.convex.cloud
BETTER_AUTH_SECRET=$(openssl rand -base64 32)

# Security
NODE_ENV=production
LOG_LEVEL=info

# CORS - your frontend domain(s)
ALLOWED_ORIGINS=https://your-domain.com,https://www.your-domain.com
```

### Deploying to Render/Railway

#### Option 1: Render

1. Push your code to GitHub
2. Create a new Web Service on Render
3. Configure:
   - **Build Command**: `bun install && bun run build:backend`
   - **Start Command**: `bun run dist/index.js`
   - **Environment**: `bun`
   - Add all environment variables from your `.env`

#### Option 2: Railway

1. Push your code to GitHub
2. Create a new project on Railway
3. Connect your GitHub repository
4. Add environment variables in Railway dashboard
5. Deploy

#### Convex Production

```bash
# Deploy Convex to production
npx convex deploy

# Update your CONVEX_URL to the production deployment
```

### Security Hardening

The following security features are enabled:

| Feature | Description |
|---------|-------------|
| Rate Limiting | 100 req/min (global), 10 req/min (auth), 50 req/min (ledger) |
| CORS | Configurable via `ALLOWED_ORIGINS` env var |
| CSP | `default-src 'self'`, `script-src 'self'`, etc. |
| HSTS | `max-age=31536000; includeSubDomains` |
| X-Frame-Options | DENY |
| Nonce Replay Protection | 5-minute TTL for used nonces |
| Session Expiry | 7 days with automatic cleanup |

```
  ▼
[Elysia Core API (Bun)] ◄──► [Protocol Parser (x402/AP2)]
  │                             
  ▼ (ConvexHttpClient)         
[Convex Database] ─────────────┐
  │                             │
  ▼                             ▼
[Auto Tax/Fee Engine]      [React Dashboard]
```

## API Endpoints

### Authentication
- `POST /v1/auth/session` - Create session with wallet signature
- `POST /v1/auth/dev/create` - Create dev agent (bypasses signature)
- `GET /v1/auth/me` - Get current agent info
- `POST /v1/auth/logout` - Invalidate session

### Ledger
- `POST /v1/ledger/transact` - Record x402/AP2 transaction (0.1% fee deducted)
- `POST /v1/ledger/prepare` - Prepare transaction (generate SIWE message)
- `GET /v1/ledger/recent` - Get recent transactions
- `GET /v1/ledger/agent/:id` - Get agent transactions
- `GET /v1/ledger/volume` - Get total transaction volume
- `GET /v1/ledger/fees` - Get total fee revenue

### Memory
- `POST /v1/memory/query` - Query financial memory (roi/spend_velocity/tax_liability)
- `GET /v1/memory/all` - Get all memory dimensions
- `GET /v1/memory/tax/stubs` - Get tax stubs
- `GET /v1/memory/tax/unpaid` - Get unpaid tax liability

### Agents
- `GET /v1/agents/:id` - Get agent by ID
- `GET /v1/agents` - List all agents
- `GET /v1/agents/stats/count` - Get active agent count

## Features

### Protocol Support
- **x402 Protocol** - Full payload parsing with expiration conditions
- **AP2 Protocol** - Task/workflow-based transactions

### Transaction Flow
1. Agent authenticates via wallet signature (or dev mode)
2. Transaction payload is validated and parsed
3. 0.1% (10 bps) fee is deducted automatically
4. Both main transaction and fee are recorded
5. Agent balance is updated
6. Real-time updates via Convex

### Financial Memory
- **ROI** - Return on investment calculations
- **Spend Velocity** - Daily/hourly spending patterns
- **Tax Liability** - Estimated tax based on 15% rate

### Real-time Dashboard
- Live transaction stream (polling every 3 seconds)
- API Playground for testing
- Memory query visualization
- Agent metrics display

## Transaction Fee

Maltheron deducts a 0.1% (10 basis points) fee on all transactions. This is recorded as a separate `fee` transaction type in the ledger.

Example: 1000 USDC transaction → 999 USDC net + 1 USDC fee

## Tech Stack

- **Runtime**: Bun
- **Backend**: Elysia + dotenv + pino
- **Database**: Convex
- **Auth**: Custom session management + viem (SIWE signatures)
- **Frontend**: Vite + React 18 + Tailwind CSS
- **Fonts**: Geist + TikTok Sans (fallback)
- **Logging**: pino (JSON in prod, pretty in dev)
